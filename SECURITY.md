# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Dokkimi, please report it responsibly.

**Email:** [security@dokkimi.com](mailto:security@dokkimi.com)

**GitHub:** You can also use [GitHub's private vulnerability reporting](https://github.com/dokkimi/dokkimi/security/advisories/new) to submit a report directly on the repository.

Please include:

- A description of the vulnerability
- Steps to reproduce
- Any relevant logs or screenshots

We will acknowledge your report within 48 hours and aim to provide a fix or mitigation plan within 7 days.

## Scope

This policy covers the Dokkimi CLI, Control Tower, and all sidecar services (interceptor, test agent, DB proxies) in this repository.

## Disclosure

We ask that you give us reasonable time to address the issue before public disclosure. We will credit reporters in the release notes unless they prefer to remain anonymous.
