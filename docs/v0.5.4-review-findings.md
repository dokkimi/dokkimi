# v0.5.4 Review Findings

Bugs found during code review of the v0.5.4 branch. All confirmed.

## 1. WORKER mountFiles silently ignored

**Severity:** High
**Files:** `shared/definition-resolver/resolve.ts:397`

The validator accepts `mountFiles` on WORKER items, but the definition resolver skips them because the guard checks `item.type !== 'SERVICE'`. Users get a clean validation but files are missing at runtime with no error.

**Fix:** Change the guard at line 397 from `item.type !== 'SERVICE'` to `item.type !== 'SERVICE' && item.type !== 'WORKER'`.

- [ ] Fixed

## 2. Content-Type header clobbers multipart boundary

**Severity:** High
**Files:** `services/test-agent/http_executor.go:234-236`

In `doAPIRequest`, user-supplied headers are applied after the auto-generated `Content-Type` (which includes the multipart boundary). If a user sets `Content-Type: multipart/form-data` in their action headers alongside `formData`, the boundary is lost and the request is unparseable.

**Fix:** Skip overwriting `Content-Type` from user headers when `formData` is in use:

```go
for key, value := range action.Headers {
    if action.FormData != nil && strings.EqualFold(key, "Content-Type") {
        continue
    }
    req.Header.Set(key, value)
}
```

- [ ] Fixed

## 3. || vs ?? credential mismatch

**Severity:** Medium-High
**Files:** `services/control-tower/src/namespace-lifecycle/builders/configmap-builder.service.ts:134`, `services/control-tower/src/namespace-lifecycle/docker/docker-database-group.service.ts:61-65`

`configmap-builder` uses `||` (logical OR) for credential fallback while `docker-database-group` uses `??` (nullish coalescing). If a user sets `dbPassword: ''`, the configmap builder treats it as falsy and falls back to `'dokkimi'`, but the database group preserves the empty string. This causes a credential mismatch between the db-proxy and the interceptor configmap.

**Fix:** Use the same operator in both files. `??` is the safer choice (preserves intentional empty strings).

- [ ] Fixed

## 4. formData/queryParams validation accepts null and arrays

**Severity:** Medium
**Files:** `shared/definition-validator/validate-tests.ts:248-255`

The validation checks `typeof x !== 'object'`, but `typeof null === 'object'` and `typeof [] === 'object'` in JavaScript. Both pass validation but fail at runtime — the Go test-agent expects `map[string]interface{}`.

**Fix:** Use the same pattern already used for `variables` validation on line 167:

```typescript
if (action.formData !== undefined && (!action.formData || typeof action.formData !== 'object' || Array.isArray(action.formData))) {
```

Same for `queryParams`.

- [ ] Fixed

## 5. formData silently dropped on type mismatch

**Severity:** Medium
**Files:** `services/test-agent/variable_context.go:149-158`

The two-value type assertion `m, ok := resolvedFormData.(map[string]interface{})` silently drops formData when it's not a map (ok=false). `resolved.FormData` stays nil and the request sends without form data. The `Params` handling at line 181 already returns an explicit error for this case.

**Fix:** Add an error return when the type assertion fails, matching the Params pattern:

```go
if m, ok := resolvedFormData.(map[string]interface{}); ok {
    resolved.FormData = m
} else {
    return resolved, fmt.Errorf("resolving formData: expected object, got %T", resolvedFormData)
}
```

Same for `queryParams` at lines 161-168.

- [ ] Fixed

## 6. formData [] double-append on array keys

**Severity:** Medium
**Files:** `services/test-agent/http_executor.go:181`

`buildFormDataBody` unconditionally appends `[]` to array field keys (`key + "[]"`). A user who names their field `tags[]` with an array value gets `tags[][]` sent to the server.

**Fix:** Check for existing `[]` suffix before appending:

```go
arrayKey := key
if !strings.HasSuffix(key, "[]") {
    arrayKey = key + "[]"
}
```

- [ ] Fixed

## 7. ReadForm temp file leak in interceptor

**Severity:** Low-Medium
**Files:** `services/interceptor/main.go:272`

`ReadForm(10<<20)` is called but `form.RemoveAll()` is never called. Multipart uploads over 10MB spill to temp files that accumulate for the lifetime of the sidecar process.

**Fix:** Add `defer form.RemoveAll()` immediately after the `ReadForm` call:

```go
if form, err := reader.ReadForm(10 << 20); err == nil {
    defer form.RemoveAll()
    // ... rest of form processing
}
```

- [ ] Fixed
